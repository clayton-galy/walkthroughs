TryHackMe Write-Ups

Cold VVars

March 16, 2023

Cold VVars is a Ubuntu web server running Apache and Samba SMB. On the website, I come across a login page that can be bypassed by using an XPATH injection. Once logged in, I am given a list of usernames and passwords. Using one of the credential pairs, I can log into the SMB service and view a “dev” share. When I upload files to the share, I can get access to the files directly from the website. I’ll upload a PHP web shell and use it to get a full shell on the box as the www-data user. On the box, I can switch to another user using the credentials found on the website. As the ArthurMorgan user, I notice a strange environment variable referencing a port that wasn’t listening on the box. I’ll use Netcat to open the port and start receiving data from a custom application. Using the custom application, I’ll drop into a VIM instance and use it to execute commands and escalate my privileges to another user on the box. This new user is running a tmux session that has a root login. I’ll attach to the tmux session to use the root shell.

Ollie

March 12, 2023

Ollie is a Ubuntu web server running Apache and a strange service on port 1337. Connecting to port 1337 manually and following the prompts returns a credential pair. I’ll use the credential pair to login to the phpIPAM instance on the website. After, I’ll utilize a remote command execution vulnerability to get a shell on the box as the www-data user. I can use the same password I found earlier to escalate to the ollie user. From there, I’ll discover a service that is running a script every few minutes as the root user. This script is writable by the ollie user and I’ll abuse this to escalate to the root user.

ConvertMyVideo

February 1, 2023

ConvertMyVideo is a Ubuntu web server running Apache. The site offers functionality to convert a YouTube video into a MP3 format. I’ll discover that the server takes user input and runs a system command with it. I’ll abuse this fact and inject my own arbitrary commands with some linux tricks to get a shell on the box as the www-data user. After, I’ll find a cronjob that is executing a script as root that I have write access to. I’ll overwrite the script to get a shell as the root user.

EnterPrize

February 1, 2023

EnterPrize is a Ubuntu web server running Apache. Running feroxbuster on the site, I’ll discover a composer.json file that reveals the site is using TYPO3. After enumerating subdomains, I’ll find where the TYPO3 instance is located. Looking around the TYPO3 files, I’ll find a backup file of a configuration file that contains an encryption key that validates parameters that are sent to the server. I’ll use this encryption key to send arbitrary parameters, leading to a deserialization vulnerability that gives me remote command execution on the server. I’ll use this vulnerability to get a shell on the box as the www-data user. On the box, I’ll find a binary that’s being regularly executed by another user. I’ll use ltrace to discover a library that the binary is using that I have write access to. I’ll create my own malicious library and overwrite the legitimate library to execute arbitrary commands and get a shell as the john user. The john user has access to a NFS share that was hiding behind a firewall. The NFS configuration shows that no_root_squash is enabled. I’ll abuse this fact to get a shell on the box as the root user.

Aratus

December 29, 2022

Aratus is a CentOS web server running FTP, Apache, and Samba SMB. I’ll get access to a public SMB share that contains many files. After downloading the files, I’ll perform a search across all of the files and discover an encrypted private SSH key. I’ll crack the password of the key with JohnTheRipper and use it to log into the box as the simeon user. On the box, I’ll notice that tcpdump has special capabilities set so I can use it to sniff network traffic to and from the box. In doing so, I’ll find a cleartext credential pair from a HTTP request and use it to login to the box as the theodore user. The theodore user has sudo privileges to run Ansible playbooks as root. I’ll modify one of the playbooks to execute arbitrary commands to get a reverse shell as the root user.

Annie

December 27, 2022

Annie is a Ubuntu server running the AnyDesk client. I’ll find a CVE for a remote code execution vulnerability in this application to get a shell on the box as the annie user. As the annie user, I’ll find the annie user’s encrypted private SSH key and crack the password with JohnTheRipper. Afterwards, I’ll discover that python has special capabilities set to allow users to change the UID in their python commands without a password. I’ll abuse this to get a shell as the root user.

BiteMe

December 27, 2022

BiteMe is a Ubuntu web server running Apache. In the JavaScript found on the login form on the website, I’ll find that syntax highlighting is enabled, allowing me to view the underlying PHP code of the site. I’ll abuse this to find an encoded password in the code that I can decode. After attempting to login with the credentials I got, I am greeted with a MFA screen. I’ll use wfuzz to fuzz the MFA code until I get a code that works. Once logged in, I’ll use the file browser to grab an encrypted private SSH key. I’ll crack the password for the key with JohnTheRipper and use the key to login to the box as the jason user. I’ll use the jason user’s sudo privileges to escalate to the fred user, who has sudo privileges to restart the fail2ban service. I’ll change the fail2ban configuration to execute a malicious command when a ban is triggered. Then I’ll restart the service, trigger the ban, and get an easy root shell.

VulnNet: Endgame

December 8, 2022 & December 20, 2022

VulnNet: Endgame features a Ubuntu web server running Apache. I’ll enumerate subdomains to find four new endpoints. One of the endpoints, the blog, uses an API to fetch the data for each post on the blog. I’ll eventually find that this API call is vulnerable to a Blind SQL injection by conditional responses. I wrote a python script to exfiltrate the data from the database, before realizing the SQL injection can be modified so I can see the responses. With this correction, I’ll dump all of the usernames and passwords from the database. Using the cleartext passwords found in the database, I can crack a password hash found in another part of the database with one of the cleartext passwords. With this special password, I can log into a TYPO3 instance as the admin user found on one of the other subdomains. Once logged in, I’ll modify some settings in the TYPO3 instance so I can upload PHP files and then upload a PHP web shell. With the web shell I’ll get a full shell as the www-data user on the box. On the box, I’ll find a FireFox profile in a user’s home directory and use it to grab a saved password. With the password, I’ll escalate my privileges to the system user. Afterwards, I’ll find that OpenSSL has all capabilities set. I’ll abuse this fact to escalate my privileges to root by setting up an SSL server that can overwrite files on the box. I’ll overwrite the shadow file with a password for the root user that I know to get a root shell on the box.

WWBuddy

August 11 & 31, 2022

WWBuddy is an Ubuntu web server running Apache. I’ll find a way to register a new user on the website which seems to be a chat application. I’ll perform a second-order SQL injection in the username of a newly registered account and reset the password for a legitimate user to one I know. Now logged in as an admin user, I get access to the website logs. I’ll create another new user with PHP code in the username. Viewing the log file executes the PHP code and I can get a shell on the server as the www-data user. On the box, I’ll read SQL log files to find another password and use it to escalate to the roberto user. The roberto user has a note in their home directory that hints at a potential password format for another user. I’ll create my own wordlist with this format to find a password for the jenny user. As the jenny user, I’ll find a SUID binary that doesn’t use the full path when executing a command. I’ll take advantage of this by polluting the PATH to get a shell as the root user.

Olympus

August 11 & 31, 2022

Olympus features a Ubuntu web server running an Apache website. After doing some enumeration of the site, I’ll find an SQL injection in a search feature on the website. I’ll use the SQL injection to dump usernames and passwords from the database. Using hashcat, I am able to crack one of the password hashes I found in the SQL database. The credential pair I found can be used to login to a chat application being hosted on a separate subdomain. Once logged in, I’ll find a way to upload PHP files to the server. However, due to the randomization of the filename, I can’t directly access my PHP file I uploaded. To get around this, I’ll use the SQL injection to pull the exact filename so I can access my PHP web shell. I’ll use the web shell to get a full shell on the box as the www-data user. On the box, I’ll find a SUID binary that’s owned by another user on the box. I’ll use the binary as it’s intended, copying one file to another location. Specifically, I’ll use the binary to copy the zeus user’s encrypted private SSH key to a directory I control. I’ll then crack the SSH key and login as the zeus user. As the zeus user, I’ll discover a PHP file that’s apparently a root backdoor. I’ll use the backdoor to get a shell as the root user.

Sea Surfer

August 7, 2022

Sea Surfer is a Ubuntu web server that is running a WordPress site. After some subdomain enumeration, I’ll find an endpoint meant for employees that takes data and generates a PDF receipt. I’ll find it’s vulnerable to XSS and turn this exploit into a SSRF by injecting an iframe. Afterwards, I’ll find out the service is vulnerable to LFI by utilizing a location header. I’ll leak the WordPress credentials and use my administrative access to get a shell on the box. On the box, I’ll abuse a cronjob that’s using a tar command with a wildcard to escalate my privileges to another user. To get root, I’ll reuse a sudo token through GDB to get command execution as root.

Undiscovered

August 3, 2022

Undiscovered is a Ubuntu web server running RiteCMS. After subdomain enumeration, I’ll find a login page and brute-force the admin credentials. With administrative access, I’ll upload a PHP web shell to eventually get a reverse shell on the box. With access, I’ll mount an NFS share on my box and get access to the contents by mimicking a user on my local machine, then abuse the “no_root_squash” configuration to get a shell as another user. I’ll then find a SUID binary that allows me to read files as another user, and I’ll use this to read a private SSH key to again escalate my privileges further. To get root, I’ll abuse special capabilities set on Vim to get a root shell.

Develpy

July 30, 2022

Develpy is a Ubuntu server running a custom service that pings a domain via Python. I’ll inject my own Python code into the application to get command execution and eventually a shell on the box. On the box, I’ll find an image that has an encoded message. I’ll decode the message and use the credentials found to escalate my privileges to another user. To get root, I’ll port forward a service running locally on the box and abuse a upload feature to get files into a directory that are being executed with a cronjob as the root user.

Spring

July 25, 2022

Spring is a Ubuntu web server running the Spring framework. I’ll find a .git directory and dump the contents to my box and analyze the web files. I’ll find a way to access the actuators to get RCE on the box and eventually a shell on the box. Once on the box, I’ll find a new password that follows a familiar format. I’ll create a custom wordlist with this password format and find the password for another user on the box. To get root, I’ll utilize a feature of the web site that seemed benign at first, but could be used to pollute a log file. By using symlinks, I am able to write the log file anywhere in the file system and I’ll abuse this by writing a public SSH key into the root user’s authenticated_keys file to get access as the root user.

Unbaked Pie

May 15, 2022

Unbaked Pie is a Ubuntu web server running a Python based website. I’ll find a cookie that is being picked, and abuse this to get RCE on the box. After getting a shell on a container, I’ll find a SQLite file with password hashes. I’ll crack one of the hashes and port forward SSH that was hiding behind a firewall and only accessible through the container. I’ll discover that a specific user is using SSH from a log file and I’ll use proxychains to brute-force this user’s SSH password. Once found, I’ll log into the target machine via SSH through the tunnel. On the box, I’ll abuse a sudo configuration to run a Python script as another user. I’ll first modify the Python script to get a shell on the box as the new user. To get root, I’ll discover the new user has a different sudo configuration that allows me to run another Python script as root and it keeps my environment variables. I’ll abuse this to hijack a library the Python script uses to get a shell as the root user.

Plotted-EMR

May 6 & 14, 2022

Plotted-EMR features a Ubuntu web server running OpenEMR and a MariaDB instance. This OpenEMR instance is running the “Multiple Sites Module” which can be abused to bypass the authentication. Since I have admin access to the MariaDB instance without supplying a password, this makes the exploit much easier to perform. Due to misconfigurations, I’ll create my own site via OpenEMR that will use the MariaDB instance to store its information in a new database that I will create. This means I can create my own administrative credentials for my site and login as an administrative user. With this access I’ll get a shell on the box via an authenticated RCE exploit. On the box, I’ll find a cronjob running rsync with a wildcard, and I’ll abuse this to escalate my privileges. To get root, I’ll use special capabilities set on ruby to get a shell as the root user.
